    II. Sumilla¶

    La asignatura de Administración de Base de Datos I pertenece al área de especialidad de ingeniería de software es de carácter teórico-práctico, de tipo obligatorio. Aporta al perfil de egreso en la competencia general de desarrollo espiritual en la sub competencia de aprendizaje del servicio enfocado en la misión, en la competencia general pensamiento de orden superior y en la competencia específica de gestión y desarrollo de software en el nivel 2. Tiene como propósito que el alumno sea capaz de diseñar correctamente modelos de Base de Datos para cualquier sistema e interactuar con ellos. Los contenidos a tratar son: Modelo conceptual de una base de datos; Diagrama Entidad Relación; Normalización, Diseño e Implementación de una base de datos. Arquitectura y componentes de una base de datos. Desarrollo de Sistemas de Base de Datos de una plataforma Cliente-Servidor.

    IV. Resultado de aprendizaje de la asignatura¶

    Resultado de aprendizaje Producto Académico
    Resultado de aprendizaje (curso): Implementa una base de datos relacional funcional para un sistema de información, modelando el dominio del negocio mediante técnicas conceptuales y relacionales, normalizando la estructura, e implementando la solución en un SGBD con SQL (DDL/DML), garantizando integridad referencial, consistencia de datos y su uso efectivo desde una aplicación web del proyecto integrador o similar. Nombre: Base de Datos Relacional del Proyecto Integrador u otro proyecto de software.
    Descripción: Diseña e implementa una base de datos relacional para el proyecto integrador o equivalente, partiendo del análisis del dominio y reglas de negocio, pasando por el modelo conceptual ER y el modelo lógico normalizado, hasta su implementación funcional en un SGBD mediante scripts SQL de definición, carga y consulta, validando su correcto funcionamiento a través de una aplicación web server-side.

    V. Unidades de aprendizaje¶

    UNIDAD 1: Diseño conceptual y lógico de bases de datos relacionales¶

    Resultado de aprendizaje Producto
    Analiza un caso de negocio para identificar entidades, atributos, relaciones y reglas del dominio, construyendo un modelo conceptual ER validado y derivando un modelo lógico relacional normalizado, coherente con los requerimientos del proyecto integrador. Nombre: Modelo conceptual ER + modelo lógico relacional validado.
    Criterios de evaluación del producto Descripción del producto
    Identificación correcta de entidades y atributos según el dominio
    Relaciones, cardinalidades y restricciones correctamente modeladas
    Uso adecuado de notación ER
    Transformación coherente ER → modelo lógico
    Normalización aplicada correctamente (1FN–3FN)
    Ausencia de inconsistencias conceptuales    		 Modela el dominio del proyecto integrador mediante un Modelo Entidad-Relación formal (entidades, atributos, relaciones, cardinalidades y restricciones), incorporando reglas de negocio y validándolo con el contexto del problema. A partir del ER, deriva un modelo lógico relacional (tablas, claves primarias y foráneas), normalizado hasta 3FN, asegurando coherencia estructural y semántica como base para su posterior implementación.

    UNIDAD 2: Implementación básica y consulta de bases de datos relacionales¶

    Resultado de aprendizaje Producto
    Implementa una base de datos relacional funcional a partir del modelo lógico, utilizando SQL DDL y DML, y desarrolla consultas SQL fundamentales para la manipulación y recuperación de datos requeridos por la aplicación web. Nombre: Base de datos relacional implementada + consultas SQL fundamentales
    Criterios de evaluación del producto Descripción del producto
    Esquema físico correctamente implementado (DDL)
    Integridad referencial operativa (PK, FK, restricciones)
    Carga de datos coherente y consistente
    Consultas SQL correctas y funcionales
    Correspondencia entre modelo lógico y base de datos implementada    		 Implementa el modelo lógico del proyecto integrador en un SGBD mediante SQL DDL (creación de tablas, tipos de datos, claves y restricciones), realiza la carga inicial de datos con SQL DML, y desarrolla consultas SQL fundamentales (SELECT con filtros, JOINs, subconsultas, vistas y agregaciones) que permitan verificar la integridad y el funcionamiento de la base de datos y soportar las operaciones requeridas por la aplicación web.

    UNIDAD 3: Programación del Lado del Servidor y Proyecto de Base de Datos (modelo + SQL)¶

    Resultado de aprendizaje Producto
    Desarrolla una lógica del negocio utilizando código de programación del lado del Servidor de Base de Datos. Integra y valida el diseño e implementación de la base de datos del proyecto integrador, demostrando su funcionamiento real mediante su uso desde la aplicación web server-side. Nombre: Base de datos funcional considerando la programación del lado del Servidor de BD e integrada a la aplicación web
    Criterios de evaluación del producto Descripción del producto
    Coherencia completa REQ → ER → modelo lógico → BD implementada
    Funcionamiento correcto de la base de datos en el SGBD
    Integración efectiva con la aplicación web (LP1 en caso lleva el curso)
    Reportes en formato csv
    Sustentación técnica clara de las decisiones de diseño e implementación    		 Consolida el diseño y la implementación de la base de datos del proyecto integrador mediante la revisión y ajuste del modelo conceptual y lógico, la implementación final en el SGBD, y la validación funcional de la base de datos integrada a la aplicación web (LP1), evidenciando operaciones CRUD, consultas/reportes, programación del lado del Servidor de BD y orientados al posterior análisis de datos.

    VI. Estrategias metodológicas¶

    N° Estrategias de enseñanza y de aprendizaje que se aplicarán en la asignatura
    1.1 Aprendizaje Basado en Problemas: Centra el aprendizaje en la resolución de problemas reales, una habilidad esencial en casi todas las profesiones. Estimula el pensamiento crítico y la colaboración, elementos clave en el desarrollo de competencias.
    1.2 Simulación: Ofrece experiencias prácticas que son esenciales para la aplicación de teorías en situaciones del mundo real.
    1.3 Estudios de caso: Desarrolla el pensamiento crítico y la toma de decisiones al analizar situaciones complejas, preparando a los estudiantes para enfrentar problemas similares en sus futuras carreras profesionales.
    1.4 Proyectos: Fomentan habilidades de investigación, gestión del tiempo y trabajo en equipo, todas cruciales en el mundo profesional
