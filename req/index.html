    Ingenieria de Requerimientos¶

    Proposito del Curso¶

    Analizar necesidades organizacionales y definir requerimientos de software mediante tecnicas de elicitacion, modelado, validacion y documentacion, aplicando estandares de ingenieria de software para garantizar trazabilidad, consistencia y alineacion entre el problema del negocio y la solucion tecnologica.

    Producto Final del Curso¶

    Especificacion de Requerimientos de Software (SRS) Documentada

    Artefacto de referencia para el Proyecto Integrador: REQ - Producto de Unidad 3.

    Incluye:

    • Identificacion y analisis de stakeholders.
    • Definicion del alcance del sistema.
    • Requerimientos funcionales.
    • Requerimientos no funcionales.
    • Reglas de negocio.
    • Historias de usuario.
    • Casos de uso.
    • Priorizacion de requerimientos.
    • Prototipos de interfaces.
    • Matriz de trazabilidad.
    • Documento SRS basado en IEEE 29148.
    • Validacion y aceptacion de requerimientos.
    • Base de trabajo para BD1, LP1 y ADS.

    Unidad 1: Descubrimiento, Elicitacion y Analisis del Problema¶

    Proposito¶

    Identificar necesidades del negocio y comprender el dominio del problema mediante tecnicas de elicitacion, analisis de stakeholders y validacion temprana de requerimientos.

    Producto¶

    Requerimientos iniciales priorizados y prototipos validados.

    Artefacto de referencia para el Proyecto Integrador: REQ - Producto de Unidad 1.
