# Sílabo LP1 2026-2

# Lenguaje de Programación I

## Producto del curso

**Sistema Web MVC Empresarial.**

Aplicación web monolítica con persistencia, control de acceso, consultas y optimización básica.

## Resultado de aprendizaje del curso

Al finalizar el curso, el estudiante desarrolla un sistema web MVC empresarial, integrando interfaces web, formularios, persistencia de datos, relaciones, consultas, control de acceso, validaciones, manejo de errores, optimización básica, pruebas funcionales y sustentación técnica.

## Unidades de aprendizaje

### Unidad 1: Fundamentos del Desarrollo Web

| Resultado de aprendizaje | Producto |
|---|---|
| Construye interfaces web interactivas mediante arquitectura web, plantillas reutilizables, formularios, JavaScript, DOM, eventos y validación del lado del cliente. | Nombre: Página web interactiva (plantillas + formularios). |

| Sesión | Contenido | Actividad práctica | Actividad autónoma |
|---|---|---|---|
| S1 | Arquitectura Web, HTTP y estructura de aplicaciones cliente-servidor. | Reconocer el flujo HTTP y estructurar el proyecto web inicial. | Revisar ejemplos de arquitectura cliente-servidor y documentar el flujo request-response del proyecto. |
| S2 | Interfaces web con HTML, CSS y plantillas reutilizables (Bootstrap). | Construir una interfaz base responsive usando plantillas reutilizables. | Mejorar la interfaz aplicando componentes Bootstrap y criterios de usabilidad. |
| S3 | Interactividad web con JavaScript, DOM, eventos y validación de formularios. | Implementar validaciones y eventos sobre formularios del proyecto. | Practicar manipulación DOM y validaciones adicionales. |
| S4 | Formularios, procesamiento de datos e interacción web, compilados. | Integrar plantillas, formularios, validaciones y procesamiento básico de datos. | Refinar el flujo de formularios y preparar evidencia del producto U1. |
| S5 | Evaluación U1. | Sustentar la página web interactiva. | Registrar observaciones y mejoras para la Unidad 2. |

### Unidad 2: Desarrollo de Aplicaciones Web MVC

| Resultado de aprendizaje | Producto |
|---|---|
| Desarrolla una aplicación web MVC empresarial aplicando rutas, controladores, servicios, ORM, repositorios, persistencia, relaciones de datos, consultas, autenticación, autorización, sesiones, validaciones y optimización básica. | Nombre: Aplicación Web MVC con persistencia, relaciones, consultas y control de acceso. |

| Sesión | Contenido | Actividad práctica | Actividad autónoma |
|---|---|---|---|
| S6 | Arquitectura MVC: rutas, controladores, servicios, ORM, repositorios y registro/consulta de una entidad simple. | Implementar un módulo MVC inicial con registro y consulta de una entidad. | Revisar separación de responsabilidades entre ruta, controlador, servicio, repositorio y vista. |
| S7 | Persistencia de datos, operaciones transaccionales y relaciones básicas (1:1 y 1:*): ORM / Nativos. | Implementar persistencia, transacciones y relaciones básicas. | Probar escenarios de creación, consulta y actualización con relaciones. |
| S8 | Relaciones de datos complejas (*:*), integridad referencial y operaciones integradas. | Implementar una relación muchos a muchos y operaciones integradas. | Verificar integridad referencial y documentar reglas de negocio asociadas. |
| S9 | Consultas, filtros, búsqueda, ordenamiento y paginación web. | Construir vistas de consulta con filtros, búsqueda, ordenamiento y paginación. | Probar combinaciones de filtros y criterios de ordenamiento. |
| S10 | Control de acceso, autenticación, autorización y gestión de sesiones. | Implementar autenticación, autorización básica y sesiones. | Revisar flujos de inicio/cierre de sesión y protección de rutas. |
| S11 | Validaciones, manejo de errores, seguimiento de solicitudes, caché y optimización de aplicaciones web. | Agregar validaciones, manejo de errores, trazas y optimización básica. | Analizar errores frecuentes, registrar mejoras y probar tiempos de respuesta. |
| S12 | Evaluación U2. | Sustentar la aplicación MVC con persistencia, relaciones, consultas y control de acceso. | Registrar observaciones para la integración final. |
