# S14 - Validación y aceptación de requerimientos

## 1. Introducción

Tiempo: 20 min.

### 1.1 Propósito

Validar el SRS final y establecer criterios de aceptación del producto integrador antes de la sustentación.

### 1.2 Resultado de aprendizaje

El estudiante aplica revisión final de completitud, consistencia, verificabilidad y aceptación del SRS.

### 1.3 Producto de sesión

SRS validado con criterios de aceptación y observaciones levantadas.

### 1.4 Motivación de la sesión

Antes de sustentar, el equipo debe comprobar que el documento representa lo construido y que las observaciones críticas fueron levantadas.

### 1.5 Ubicación en el curso

Esta sesión continúa el desarrollo progresivo del proyecto integrador y aporta evidencias para el producto de la unidad correspondiente.

## 2. Explica

Tiempo: 25 min.

### 2.1 Conceptos clave

- Validación final
- Aceptación
- Completitud
- Consistencia
- Verificabilidad
- Criterio de aceptación
- Acta de revisión
- Observación
- Levantamiento

### 2.2 Arquitectura de la sesión

`mermaid
flowchart TB
    A[Insumos del proyecto] --> B[Trabajo de la sesión]
    B --> C[Artefacto técnico]
    C --> D[Validación o prueba]
    D --> E[Integración REQ-BD1-LP1]
`

### 2.3 Flujo de trabajo

1. Revisar SRS
2. Aplicar checklist
3. Comparar con BD1
4. Comparar con LP1
5. Registrar observaciones
6. Levantar correcciones
7. Preparar acta
8. Cerrar versión final

### 2.4 Errores frecuentes y diagnóstico

| Problema | Causa probable | Solución |
|---|---|---|
| El avance no se conecta con el proyecto | Se trabajó como ejercicio aislado | Relacionar cada artefacto con requerimientos, datos o módulo web |
| La evidencia no permite verificar el trabajo | Faltan capturas, tablas o explicación | Agregar evidencias breves y legibles |
| Se cambian decisiones sin registrar | Falta control de cambios | Documentar observación, decisión y responsable |
| La integración queda débil | No se revisaron los otros cursos | Usar el SRS, modelo de datos o aplicación como referencia |

## 5. Cierre evaluativo

Tiempo: 20 min.

### 5.1 Resultados esperados

- Producto de sesión construido.
- Evidencia individual organizada.
- Relación con el proyecto integrador explicada.
- Pendientes registrados para la siguiente sesión.

### 5.2 Evidencia del producto de sesión

SRS validado con criterios de aceptación y observaciones levantadas.

### 5.3 Preguntas de defensa y reflexión

1. ¿Qué observación crítica apareció?
2. ¿Qué requisito fue aceptado?
3. ¿Qué evidencia de BD1 confirma el SRS?
4. ¿Qué evidencia de LP1 confirma el SRS?
5. ¿Qué queda como mejora?

### 5.4 Rúbrica de evaluación

| Dimensión | Peso | 3 - Logro destacado | 2 - Logro | 1 - Proceso | 0 - Inicio | Puntuación obtenida |
